1  Unicidad

El indicador de unicidad busca mostrar que ningún evento, persona o registro está registrado más de una vez. A través de la Identificación de registros únicos en la base de datos del sistema utilizando variables de identificación o combinaciones de variables.

Una de las principales características de los RNVe es que permiten realizar el seguimiento individual de las personas. Para esto es importante que las personas no se encuentren duplicadas en el registro, esto podría llevar a un error en el registro de una vacuna o analizar de manera equivocada el estado de vacunación. El objetivo de este indicador es identificar el número de individuos duplicados en el registro con el fin de realizar el proceso de deduplicación.

1.1 Metodología

Los análisis de unicidad tienden a requerir de conocimiento de la base de datos y creatividad. En un mundo ideal el análisis de unicidad consistiría en:

  1. Determinar que tipo de unicidad queremos obtener:
  • Unicidad de registro de evento de vacunación
  • Unicidad de persona
  1. Identificar las variables de clave para diferenciar un registro de otro en la base de datos.

  • Estas dependen del sistema pero pueden ser algunas de las siguientes o combinaciones de las mismas:

    • ID único de registro

    • Fecha de nacimiento

    • ID del registro nacional de las personas

    • Evento de vacunación y número de lote

1.1.1.1 Unicidad de Registro

Vemos que al utilizar la variable de ID de vacunación parece que todos los registros son únicos. Sin embargo si filtramos sin ella encontramos que hay registros duplicados

#en otras variables
data_duplicates <- data %>% 
  get_dupes(-idvacunacion) #¿qué hace el "-"?

datatable(data_duplicates %>% select(-nombrecompleto))

1.1.1.2 Unicidad de Persona

Para determinar personas únicas no necesitamos las variables de registro de vacunación, solo estas que puedan ser útiles para determinar que alguien es distinta o distinto de otra persona.

Estas pueden ser:

  • Fecha de nacimiento

  • Número de identificación del registro nacional de las personas

  • Ubicación geográfica

  • Otras variables según información disponible en RNVe

Filtramos con la función get_dupes para las variables de interés

# Obtengamos los registros duplicados
data_duplicates <- data %>% 
  get_dupes(nombrecompleto, fechanacimiento, sexo, vacuna, esquema) 

datatable(data_duplicates %>% select(-nombrecompleto))

1.1.1.3 Visualizando los registros duplicados por departamento

# Hagamos un gráfico de barras con la cantidad de duplicados por departamento


data_grafico <- data_duplicates %>% 
  select(-idvacunacion, -fecha_creacion, -fecha_modificacion) %>% 
  distinct() %>% #obtenemos solo un registro por duplicado
  group_by(dpt_nombre) %>% #agrupamos por departamento
  tally() %>% #contamos cuantos registros hay por fila
  arrange(desc(n)) %>%  #¿Se recuerdan que hace arrange? 
  mutate(dpt_nombre = factor(dpt_nombre, levels=dpt_nombre))

# En este punto podemos obtener un registro duplicado. Nos interesa
# saber cuantos errores ocurrieron en cada departamento, no cuantas veces
# ocurrió el mismo error

plot <- ggplot(data_grafico, aes(x=dpt_nombre, y=n ,fill=dpt_nombre))+
  geom_bar(stat = "identity")
